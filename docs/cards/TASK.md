# ⚙️ Task / Chore

## Objetivo

Utilize este template para registrar tarefas técnicas, configurações, melhorias de infraestrutura, manutenção do projeto ou atividades que não adicionam funcionalidades diretamente ao usuário.

---

## Descrição

Descreva claramente a tarefa que deverá ser realizada.

---

## Estado Atual

Explique como o projeto está atualmente em relação à tarefa proposta.

---

## Objetivo

Descreva o resultado esperado após a conclusão da Task.

---

## Escopo

### ✅ Inclui

Liste tudo que faz parte desta Task.

Exemplo:

- Configurar o CodeRabbit.
- Validar funcionamento em Pull Requests.
- Atualizar documentação.

### ❌ Não Inclui

Liste explicitamente o que não faz parte desta Task.

Exemplo:

- Alterações em regras de negócio.
- Implementação de novas funcionalidades.
- Correção de Bugs não relacionados.

---

## Dependências

Liste outras tarefas ou funcionalidades que precisam estar concluídas antes desta Task (caso existam).

---

## Critérios de Aceite

- [ ] Objetivo da Task concluído.
- [ ] Escopo atendido.
- [ ] Nenhum impacto negativo identificado.
- [ ] Documentação atualizada (quando necessário).

---

## Definition of Ready (DoR)

- [ ] Objetivo definido.
- [ ] Escopo documentado.
- [ ] Dependências identificadas (quando existirem).

---

## Definition of Done (DoD)

- [ ] Implementação concluída.
- [ ] Build executando sem erros.
- [ ] Lint sem erros.
- [ ] Configuração validada.
- [ ] Documentação atualizada (quando necessário).
- [ ] Pull Request aprovado.

---

## Observações

Informações adicionais relevantes para a execução da Task.

---

## Exemplo

**Descrição**

Integrar o CodeRabbit ao repositório para realizar revisões automáticas de Pull Requests.

**Estado Atual**

As revisões são realizadas apenas manualmente.

**Objetivo**

Automatizar parte do processo de Code Review utilizando o CodeRabbit.

**Escopo**

### ✅ Inclui

- Instalar o CodeRabbit.
- Validar funcionamento.
- Atualizar documentação.

### ❌ Não Inclui

- Alterações no fluxo de desenvolvimento.
- Mudanças na arquitetura da aplicação.

**Dependências**

- Repositório hospedado no GitHub.
