
# Card Templates

Este diretório contém os templates oficiais utilizados para criação de **Issues** do Projeto Carrinho.

O objetivo destes modelos é padronizar a criação dos Cards, garantindo consistência na documentação, rastreabilidade das tarefas e um fluxo de desenvolvimento organizado.

## Categorias

| Categoria | Descrição |
| ---------- | --------- |
| 🐞 **Bug** | Correção de defeitos ou comportamentos inesperados. |
| 🚀 **Feature** | Implementação de uma nova funcionalidade. |
| ⚙️ **Task / Chore** | Tarefas técnicas, infraestrutura, configurações e manutenção do projeto. |
| ✨ **Enhancement** | Evolução ou melhoria de uma funcionalidade já existente. |
| 🔨 **Refactor** | Melhorias internas no código sem alteração de comportamento. |
| 📚 **Documentation** | Criação ou atualização de documentação do projeto. |
| 🧪 **Test** | Implementação ou melhoria de testes automatizados. |
| 🔒 **Security** | Melhorias relacionadas à segurança da aplicação. |
| 🔬 **Research / Spike** | Estudos, pesquisas ou provas de conceito antes de uma implementação. |

## Objetivo

Cada categoria possui um template específico contendo apenas as informações necessárias para aquele tipo de trabalho, evitando documentação desnecessária e mantendo os Cards objetivos.

## Convenções

- Utilize sempre o template correspondente ao tipo de trabalho.
- Mantenha os Cards claros e objetivos.
- Sempre que possível, inclua contexto suficiente para que qualquer desenvolvedor consiga compreender a tarefa.
- Evite misturar diferentes tipos de trabalho em um único Card.

---

Os templates deste diretório servem como referência para a criação de Issues e poderão evoluir conforme as necessidades do projeto.
